ğŸ§  How Node.js Works (Step-by-Step)

JavaScript Code Run Hoti Hai
Tumhara JS code V8 Engine me execute hota hai.
Example:

console.log("Hello Node.js");


Ye line directly V8 engine ke through chal jaati hai.

Async Task Detect Hota Hai
Jab koi asynchronous operation milta hai (like fs.readFile() ya setTimeout()),
Node.js us task ko libuv ke through background thread pool me bhej deta hai.

Main Thread Free Rehta Hai
Jab tak async kaam complete nahi hota, event loop next tasks execute karta rehta hai.
Isliye Node.js non-blocking hai.

Callback Queue
Jab background task complete hota hai, uska callback queue me chala jata hai.

Event Loop Executes Callback
Event loop queue se callback uthata hai aur execute karta hai.
Is process ko hi hum kehte hain Event-driven programming.

ğŸ” Event Loop Cycle

Event loop continuously ye steps follow karta hai:

Timers Phase â†’ setTimeout, setInterval callbacks run karta hai.

I/O Callbacks Phase â†’ Background I/O callbacks handle karta hai.

Idle / Prepare Phase â†’ Internal work of Node.js.

Poll Phase â†’ New I/O events ke liye wait karta hai.

Check Phase â†’ setImmediate() callbacks execute karta hai.

Close Callbacks Phase â†’ Close event callbacks (like socket.on('close')).

ğŸ§µ Thread Pool in libuv

By default, thread pool me 4 threads hote hain.

Ye threads background me heavy kaam karte hain jaise:

File system operations

Compression (zlib)

DNS lookups

Crypto functions

Example:

const fs = require("fs");

console.log("Start");

fs.readFile("data.txt", "utf8", (err, data) => {
  console.log("File Read Complete");
});

console.log("End");


Output:

Start
End
File Read Complete


Explanation:

fs.readFile background me chala gaya (libuv thread pool).

Event loop next task (console.log("End")) run karta raha.

Jab file read complete hua, callback queue me gaya, phir event loop ne execute kiya.

ğŸ”’ Advantages of Node.js

High performance due to V8 engine

Handles many clients at once (asynchronous I/O)

Huge community and npm support

Same language (JS) for frontend + backend

Callback hell (though promises & async/await solve this).
